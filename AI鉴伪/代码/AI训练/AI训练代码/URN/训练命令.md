# URN 训练命令

目录：

```powershell
cd C:\Users\admin\Desktop\SE\Project\代码\AI训练\AI训练代码\URN
```

说明：

- 下面每组命令都分成两行：先激活 `causal` 环境，再启动训练。
- 当前命令不会修改源码文件，只是在运行时临时覆盖训练集、测试集和日志目录。
- 训练完成后，需要把对应日志目录里的 `checkpoint_*.pkl` 手动复制到 AI 服务目录并重命名。
- 这些 `train_*.txt` / `test_*.txt` 里包含大量 Linux 绝对路径；如果本机没有对应数据路径，训练会在读取数据时报错。

## 1. Splicing / Coarse_v2

目标产物：

- `Coarse_v2.pkl`

命令：

```powershell
conda activate causal
python -c "from train import train_net; from factory.model_factory import get_model; from configs.coarse_bio_hyper_para import hp as base_hp; from configs.config import config_dict as config; from copy import deepcopy; hp=deepcopy(base_hp); hp.data.train='train_splicing'; hp.data.test='test_splicing'; config.logs_dir=r'C:\Users\admin\Desktop\SE\Project\代码\AI训练\AI训练代码\URN\logs\splicing'; config.data_txt_dir=r'C:\Users\admin\Desktop\SE\Project\代码\AI训练\AI训练代码\URN\data'; config.bio_data_txt_dir=r'C:\Users\admin\Desktop\SE\Project\代码\AI训练\AI训练代码\URN\data'; net=get_model(hp).cuda(); train_net(net,hp,config)"
```

训练完成后建议映射：

- `logs\splicing\checkpoint_*.pkl` -> `AI服务\AI服务器代码\method\urn\weight\Coarse_v2.pkl`
