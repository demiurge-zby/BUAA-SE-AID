# BUAA-SE-AID

这是一个面向学术图像鉴伪与人工复核协同的系统项目，整体由前端用户端、前端管理端、后端业务服务和独立 AI 推理服务组成。系统面向图片或文档中的图像内容，支持上传、AI 检测、人工审核、结果追踪、通知联动和管理分析。

## 项目功能

- 用户端：用户登录、文件上传、图片抽取、检测任务提交、检测结果查看、历史记录查询、人工复核申请、通知查看、个人资料管理。
- 管理端：管理员登录、用户与组织管理、人工审核流程处理、文件管理、日志查看、统计分析看板。
- 后端：提供统一 API，负责用户认证、组织与邀请码管理、文件与图片管理、检测任务编排、人工审核流转、通知推送、日志记录、报表生成。
- AI 服务：负责图像鉴伪推理，包含单图/多图检测流程，并保留训练与评测相关代码。

## 整体架构

### 1. 前端层

- `AI鉴伪/代码/前端/前端用户端`：基于 Vue 3 + Vite + Vuetify 的用户侧界面。
- `AI鉴伪/代码/前端/前端管理端`：基于 Vue 3 + Vite + Vuetify 的管理侧界面。
- 两个前端都通过 `VITE_API_URL` 对接后端 API；用户端还包含通知、任务详情、审核页面等业务页面。

### 2. 业务后端层

- `AI鉴伪/代码/后端/后端代码`：基于 Django 的主业务服务。
- 使用 Django REST Framework 提供接口，使用 JWT 做登录态管理。
- 使用 Channels 处理通知相关的 WebSocket 能力。
- 使用 Celery 处理异步检测任务，Redis 用作消息中间件。
- 核心业务模型包括用户、组织、邀请码、文件管理、检测任务、检测结果、人工审核请求、人工审核记录、通知和操作日志。

### 3. AI 推理层

- `AI鉴伪/代码/AI服务/AI服务器代码`：独立的推理与训练代码目录。
- 目录内包含检测主入口、单图/多图 pipeline、传统方法与 LLM 相关方法、训练与评测脚本。
- 后端通过任务调度或服务协作的方式调用 AI 推理能力，完成检测结果回传。

### 4. 部署与运行关系

- 前端负责页面展示与交互。
- 后端负责接口、权限、任务、审核和通知。
- AI 服务负责实际图像鉴伪计算。
- Redis、Celery、MySQL 共同支撑异步任务和数据持久化。

## 使用方法

### 环境说明

- 如需运行 Python 相关代码，先执行 `conda activate se`。
- 服务器端 MySQL 服务已启动，但当前说明中不要求连接华为云数据库。
- 后端部署目录可参考 `/root/se/data`。
- 当前已有 3 个 `tmux` 进程在运行，可通过 `tmux a -t ai`、`tmux a -t cpu`、`tmux a -t backend` 连接查看。

### 前端用户端

目录：`AI鉴伪/代码/前端/前端用户端`

1. 安装依赖：

```bash
npm install
npm install marked
npm install axios
```

2. 在 `.env` 中配置后端地址：

```env
VITE_API_URL=http://122.9.32.72:8000
```

3. 启动开发环境：

```bash
npm run dev
```

### 前端管理端

目录：`AI鉴伪/代码/前端/前端管理端`

1. 安装依赖：

```bash
npm install
```

2. 在 `.env` 中配置后端地址，通常形式如下：

```env
VITE_API_URL=http://后端地址:端口
```

3. 启动开发环境：

```bash
npm run dev
```

### 后端

目录：`AI鉴伪/代码/后端/后端代码`

常规运行入口为：

```bash
python manage.py runserver
```

生产或联调场景下，代码中还包含 Django、uWSGI、Nginx、Redis、Celery、WebSocket 的组合部署方式。后端负责对接前端请求、写入数据库、创建异步检测任务，并与 AI 服务协同。

### AI 推理服务

目录：`AI鉴伪/代码/AI服务/AI服务器代码`

- 推理主入口：`main.py`
- 训练脚本：`train.py`
- 评测脚本：`eval.py`
- 单图推理脚本：`infer.py`

典型命令示例：

```bash
python train.py
python eval.py
python infer.py
```

AI 推理服务建议部署在具备 GPU 的环境中。

## 后端核心模块说明

- `core/models.py`：定义用户、组织、检测任务、检测结果、人工审核、通知、日志等核心数据模型。
- `core/urls.py`：集中注册业务接口，包括用户、上传、检测、审核、组织、通知、管理分析等 API。
- `core/views/`：按照用户、上传、检测、审核、组织、管理、通知等领域拆分视图逻辑。
- `fake_image_detector/settings.py`：后端配置中心，涉及数据库、JWT、Channels、Celery、静态与媒体资源等配置。
